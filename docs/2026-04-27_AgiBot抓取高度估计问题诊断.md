- **预测方法**: Flow Matching with 16步去噪

## 2. 根本原因分析

### 2.1 动作归一化偏差

**位置**: `groot/vla/data/transform/state_action.py:84-200`

系统使用`Normalizer`类将关节角度归一化到[-1, 1]范围：

```python
# 归一化公式 (min_max模式)
normalized = 2 * (x - min) / (max - min) - 1

# 反归一化公式
x = (normalized + 1) / 2 * (max - min) + min
```

**问题**：
- 如果训练数据的`min`和`max`统计量不准确，会导致系统性偏差
- 特别是高度相关的关节（关节4），如果统计量偏高，反归一化后会高估高度

### 2.2 Flow Matching噪声调度问题

**位置**: `groot/vla/model/dreamzero/action_head/wan_flow_matching_action_tf.py:652-704, 1160-1167`

系统支持解耦的视频-动作噪声采样：

```python
# 训练时 (第655-686行)
if self.config.decouple_video_action_noise:
    # 视频: Beta分布，偏向高噪声
    video_noise_ratio = self.video_beta_dist.sample([...])
    # 动作: 独立的均匀分布
    timestep_action_id = torch.randint(0, self.scheduler.num_train_timesteps, ...)

# 推理时 (第1160-1167行)
if self.config.decouple_inference_noise:
    video_final_noise = self.config.video_inference_final_noise  # 默认0.8
    # 视频保持在较高噪声水平，动作完全去噪
```

**问题**：
- 推理时视频保持在80%噪声水平（`video_inference_final_noise=0.8`）
- 这可能导致视觉特征不够清晰，影响动作预测精度
- 特别是对于需要精确定位的抓取任务

### 2.3 关节空间误差放大

**问题**：
- 系统在**关节空间**预测动作，而高度是**笛卡尔空间**的概念
- 小的关节角度误差（如0.01弧度）在末端执行器上会被放大
- 对于7自由度机械臂，末端位置对某些关节（特别是肩部和肘部关节）非常敏感

**示例计算**：
```
假设手臂长度 L = 0.5m
关节角度误差 Δθ = 0.01 rad
末端高度误差 Δz ≈ L × sin(Δθ) ≈ 0.5 × 0.01 = 0.005m = 5mm
```

对于多关节累积误差，实际偏差可能达到2-3cm。

## 3. 解决方案

### 方案1: 添加Z轴偏移校准参数 ⭐ **推荐优先尝试**

**难度**: ⭐☆☆☆☆ (最简单)  
**效果**: ⭐⭐⭐☆☆ (快速验证)  
**实施时间**: 5-10分钟

#### 实施步骤

在 `robot_live_client.py` 中添加高度补偿参数：

```python
# 在文件顶部添加常量 (约第68行附近)
HEIGHT_COMPENSATION_LEFT_ARM_JOINT_4 = -0.03  # 负值降低高度，单位：弧度
HEIGHT_COMPENSATION_RIGHT_ARM_JOINT_4 = -0.03

# 在应用动作的函数中添加补偿 (搜索 "policy_to_sdk_arm" 或动作执行部分)
def apply_height_compensation(arm_joints: np.ndarray, is_left_arm: bool) -> np.ndarray:
    """Apply height compensation to arm joint positions."""
    compensated = arm_joints.copy()
    if is_left_arm:
        compensated[4] += HEIGHT_COMPENSATION_LEFT_ARM_JOINT_4
    else:
        compensated[4] += HEIGHT_COMPENSATION_RIGHT_ARM_JOINT_4
    return compensated
```

#### 调参指南

1. **初始值**: 从 `-0.03` 弧度开始（约-1.7度）
2. **观察效果**: 
   - 如果仍然空抓，增加补偿量（更负）：`-0.05`
   - 如果抓得太低，减少补偿量：`-0.01`
3. **精细调整**: 以0.01弧度为步长调整
4. **验证**: 在多个不同高度的物体上测试

#### 优缺点

✅ **优点**:
- 实现简单，5分钟内可以验证
- 不需要重新训练模型
- 可以快速迭代调参

❌ **缺点**:
- 治标不治本，只是添加固定偏移
- 可能对不同高度的物体效果不一致
- 需要手动调参

1. **重新计算统计量**：

### 方案5: 增加高度相关关节的训练损失权重 ⚠️ **需要重新训练**

**难度**: ⭐⭐⭐⭐☆  
**效果**: ⭐⭐⭐⭐⭐  
**实施时间**: 数小时到数天（取决于训练规模）

#### 实施步骤

修改 `groot/vla/model/dreamzero/action_head/wan_flow_matching_action_tf.py` 的训练代码：

```python
# 在 forward 方法中，约第758行附近
# 原代码：
action_loss_per_sample = torch.nn.functional.mse_loss(
    action_noise_pred.float(), training_target_action.float(), reduction='none'
) * action_mask

# 修改为：
action_loss_per_sample = torch.nn.functional.mse_loss(
    action_noise_pred.float(), training_target_action.float(), reduction='none'
) * action_mask

# 添加关节权重
joint_weights = torch.ones_like(action_loss_per_sample)
joint_weights[:, :, 4] *= 2.0   # 左臂关节4 (高度相关)
joint_weights[:, :, 11] *= 2.0  # 右臂关节4 (高度相关)
action_loss_per_sample = action_loss_per_sample * joint_weights
```

#### 优缺点

✅ **优点**:
- 从根本上提高模型对高度的预测精度
- 长期效果最好
- 可以泛化到不同场景

❌ **缺点**:
- 需要重新训练模型（时间成本高）
- 需要训练数据和计算资源
- 可能影响其他关节的预测精度

## 7. 常见问题

### Q1: 为什么不直接在笛卡尔空间预测？

A: DreamZero在关节空间预测有以下优势：
- 避免逆运动学求解的复杂性
- 更容易处理关节限制
- 训练数据直接来自关节编码器

但缺点是小的关节误差会被放大到末端执行器。

### Q2: 补偿参数应该设置多大？

A: 建议从-0.03弧度开始，根据实际效果调整：
- 如果仍然空抓：增加到-0.05
- 如果抓得太低：减少到-0.01
- 精细调整：以0.005为步长

### Q3: 是否需要同时调整多个关节？

A: 建议先只调整关节4（肘部），因为它对高度影响最大。如果效果不佳，可以考虑同时调整关节2（肩部）。
