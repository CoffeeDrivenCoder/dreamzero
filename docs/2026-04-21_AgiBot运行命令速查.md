# AgiBot 最新运行命令

日期：2026-04-21

这份文档用于留存当前联调阶段可直接使用的 DreamZero AgiBot server / client 启动命令。

当前约定：

- 服务器代码目录：`/home/user/wangk/dreamzero`
- 客户端代码目录：`/home/ke/dreamzero`
- 模型服务端口：`9443`
- 服务端使用 `0,1` 号 GPU

## 当前状态说明

- 服务器端 `eval_utils/policy_server.py` 已关闭 websocket heartbeat：
  - `ping_interval=None`
  - `ping_timeout=None`
- 当前推荐服务端命令默认使用 `ATTENTION_BACKEND=FA2`
- 如果 `FA2` 再出现 attention 兼容问题，可回退到 `ATTENTION_BACKEND=torch`
- 当前客户端基线配置：
  - `5 FPS`
  - `4` 帧 observation history
  - arm trajectory interval `0.008`
  - arm close timeout `0.6`

## 服务器启动命令

在服务器机器执行：

```bash
cd /home/user/wangk/dreamzero

unset http_proxy https_proxy HTTP_PROXY HTTPS_PROXY all_proxy ALL_PROXY
unset ws_proxy wss_proxy WS_PROXY WSS_PROXY

export HF_HUB_OFFLINE=1
export PYTORCH_NVML_BASED_CUDA_CHECK=1
export NO_ALBUMENTATIONS_UPDATE=1
export ATTENTION_BACKEND=FA2

CUDA_VISIBLE_DEVICES=0,1 /home/user/miniconda3/envs/dreamzero/bin/python -m torch.distributed.run --standalone --nproc_per_node=2 socket_test_optimized_AR.py \
  --port 9443 \
  --model-path /home/user/wangk/checkpoints/DreamZero-AgiBot \
  --embodiment-tag agibot \
  --enable-dit-cache
```

如果 `FA2` 报错，改成：

```bash
export ATTENTION_BACKEND=torch
```

然后重新执行同一条 server 命令。

## 客户端启动命令

在机器人客户端机器执行：

```bash
cd /home/ke/dreamzero

python robot_live_client.py \
  --host 223.167.85.178 \
  --port 9443 \
  --prompt "Grasp the green can." \
  --sdk-arm-order left_right \
  --apply-actions \
  --observation-fps 5 \
  --observation-history 4 \
  --arm-trajectory-interval 0.008 \
  --arm-close-timeout 0.6
```

## 仅推理验证命令

如果这轮只想验证 server 推理链路和 websocket 是否稳定，不让机器人执行动作，可去掉 `--apply-actions`：

```bash
cd /home/ke/dreamzero

python robot_live_client.py \
  --host 223.167.85.178 \
  --port 9443 \
  --prompt "Grasp the green can." \
  --sdk-arm-order left_right \
  --observation-fps 5 \
  --observation-history 4
```

## 客户端代码版本提醒

当前对话里已经在服务器工作区的 `robot_live_client.py` 中实现了“row-synced gripper”逻辑。

如果客户端机器上的 `/home/ke/dreamzero/robot_live_client.py` 还没有同步这份修改，那么：

- 启动命令仍然可以使用本文档中的命令
- 但实际夹爪执行行为仍取决于客户端机器上的本地文件版本

建议在关键联调前确认客户端日志里是否出现如下字样：

```text
Executed move_arm dual-arm trajectory with row-synced gripper
```

如果出现这句，说明客户端已经运行到逐行同步夹爪的版本。

## 运行前检查

- 确认服务器 `0,1` 号 GPU 已空闲
- 确认 server 已用最新代码重启
- 确认 client 连接的 `host` 和 `port` 正确
- 如出现 `1011 keepalive ping timeout`，优先检查 server 是否仍在运行旧进程
