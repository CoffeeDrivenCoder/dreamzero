# 2026-04-28 AgiBot 真机调试记录

本文记录今天 AgiBot 真机推理与执行调试中遇到的主要问题、判断和处理方式，便于后续复现实验和继续优化。

## 1. 推理速度明显变快

**现象**

服务端日志显示单次推理约 2.6-3.0 秒：

```text
Inference Time: Total 2.6~3.0 seconds
DIT Compute Steps 4 steps
```

**原因判断**

主要原因是 action head 已经和 checkpoint 配置对齐，实际使用 `num_inference_timesteps=4`。此前代码中存在硬编码 16-step 的问题，和 AgiBot checkpoint 的 4-step 配置不一致。

**处理**

- 将 action head 的 `num_inference_steps` 改为读取 checkpoint/config 中的 `num_inference_timesteps`。
- 对 `num_inference_steps <= 4` 的情况，不再套用 16-step 的 DiT mask 跳步逻辑，确保 4 个 step 都真实执行。

## 2. 服务器缓存状态需要显式清理

**现象**

多次任务/不同 prompt 连续运行后，模型可能出现动作不稳定或历史状态错位风险。

**原因判断**

DreamZero action head 内部维护 KV cache、图像特征、语言状态和 `current_start_frame`。如果 session 切换或客户端断开时不清理，后续推理可能继承上一轮任务的状态。

**处理**

- 在 action head 中增加 `reset_inference_cache()`。
- 服务端在 session 切换、reset、客户端断开时广播 cache reset 信号。
- rank0 和 worker 都执行 cache reset，降低分布式推理状态不一致风险。

## 3. 生成视频保存

**需求**

需要保存模型生成的视频结果，用于观察模型预测和失败原因。

**处理**

- 服务端新增 `--video-save-mode`，支持：
  - `none`：不保存；
  - `first`：每个 chunk 只保存首帧；
  - `full`：保存完整生成视频 chunk。
- AgiBot 当前建议使用 `--video-save-mode full`。
- 视频保存目录固定为：

```text
/home/user/wangk/dreamzero/video_rollout
```

- 文件名使用时间戳和帧数，例如：

```text
20260428_165605_000008_f429.mp4
```

## 4. `current_start_frame >= local_attn_size` 日志

**现象**

服务端出现：

```text
current_start_frame >= local_attn_size, reset current_start_frame to 0
```

**原因判断**

这不是报错，而是因果视频模型的局部注意力窗口达到上限后，自动重置当前起始帧。当前 `local_attn_size` 有限，连续若干个 chunk 后会触发该逻辑。

**影响**

- 可能导致周期性丢失一部分历史上下文；
- 可能带来轻微延迟波动；
- 不是当前最大瓶颈，但需要在分析长任务连续性时留意。

**建议**

先记录观察，不作为首要修改目标。优先保证推理步数、cache reset、观测/执行闭环正确。

## 5. 异步/并行预取推理效果变差

**现象**

开启 speculative inference 后，机器人执行速度看似更连续，但任务效果变差，出现抖动、不推进或动作怪的问题。

**原因判断**

当前实现是在当前动作 chunk 执行中途采集观测，并提前请求下一段动作。但服务端 action head 是有状态 causal cache，提前推理会推进服务器端 cache。

如果 speculative 结果被拒绝、过期，或与真实 chunk 边界状态不一致，服务端 cache 仍可能已经被污染/推进，导致后续同步推理和真实机器人历史错位。

**处理建议**

- 当前阶段不建议开启：

```text
--enable-speculative-inference
```

- 回到同步闭环推理，优先保证动作质量。
- 若后续继续做异步，需要支持以下机制之一：
  - speculative 推理不写入主 cache；
  - 拒绝 speculative 结果后可以 rollback cache；
  - speculative 分支使用独立 cache；
  - 每次 speculative 后强制 reset，但这会损失 causal history。

## 6. 夹爪抓纸杯过硬

**现象**

纸杯抓取任务中，夹爪直接闭合到 1.0 容易把纸杯捏扁。

**处理**

客户端增加 `--gripper-closed-value` 参数，用于控制模型判定闭合时实际发送的夹爪值。

实验值：

```text
--gripper-closed-value 0.7
```

后续可继续测试 0.6、0.7、0.8 等值，观察是否能在不压扁纸杯的情况下稳定抓取。

## 7. 平滑动作导致执行观感异常

**现象**

开启 Savitzky-Golay 平滑后，机器人有时表现为抖动但不明显移动。

**原因判断**

平滑可能改变模型原始动作序列的局部幅度和时序，尤其在直接执行 48 行 action 的模式下，可能削弱关键动作变化。

**处理**

当前推荐关闭动作平滑：

```text
--action-smoothing none
```
